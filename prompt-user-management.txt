### 2. Add User Management Dashboard with Roles & Permissions
Build a complete user management system in the CRM:

#### Backend (server/)
- Create `server/userManagement.ts` with Express routes:
  - `GET /api/admin/users` — list all users (admin only)
  - `POST /api/admin/users` — create user with role
  - `PUT /api/admin/users/:id` — update user role/permissions  
  - `DELETE /api/admin/users/:id` — delete user
  - `POST /api/admin/users/:id/deactivate` — suspend user
- Add roles table to SQLite (`server/db.ts`):
  ```sql
  CREATE TABLE IF NOT EXISTS users (
    id TEXT PRIMARY KEY,
    uid TEXT UNIQUE,  -- Firebase UID
    name TEXT NOT NULL DEFAULT '',
    email TEXT,
    phone TEXT DEFAULT '',
    role TEXT DEFAULT 'user',  -- 'admin', 'manager', 'technician', 'user'
    permissions TEXT DEFAULT '{}',  -- JSON object for granular permissions
    active INTEGER DEFAULT 1,
    created_at TEXT DEFAULT (datetime('now')),
    updated_at TEXT DEFAULT (datetime('now'))
  );
  ```
- Add permission check middleware: `requireRole(roles: string[])` and `requirePermission(permission: string)`
- Update `auth.ts`: After Firebase token verification, lookup the user in the local DB to get their role

#### Frontend (src/)
- Create `src/pages/AdminUsers.tsx` — user management page with:
  - Table of all users (name, email, role, status, last login)
  - Add user form (name, email, role dropdown)
  - Edit user role (dropdown: admin/manager/technician/user)
  - Deactivate/Activate toggle
  - Search/filter
- Create `src/components/UserRoleBadge.tsx` — colored badge showing role
- Add route `/admin/users` in `src/App.tsx` (protected for admin role only)
- Add "User Management" link in sidebar/nav (visible to admins only)
- Add permission-based UI hiding (e.g., hide delete buttons for non-admins)
