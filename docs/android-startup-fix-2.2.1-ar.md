# إصلاح بدء تشغيل BreeXe Connect 2.2.1

يعالج هذا الإصدار عدم فتح حزمة `2.2.0` على جهاز الاختبار.

- أعيد إصدار التطبيق بصيغة `debug` نفسها التي استُخدمت في حزمة `2.1.3` العاملة، وبالتوقيع نفسه لقبول التحديث فوقها.
- رُفع رقم البناء إلى `11` حتى يقبل أندرويد التثبيت فوق `2.2.0` رقم `10`.
- لم يعد فشل تحميل مكتبة SQLCipher الأصلية يغلق عملية التطبيق كاملة.
- أصبحت قراءة قاعدة Room والطابور محمية؛ يعرض التطبيق رسالة تشخيص واستعادة بدل الانهيار عند فتح الشاشة.
- لا يحذف التحديث بيانات التطبيق أو رمز الربط أو الأحداث المحلية.

## التحقق

- `testDebugUnitTest`
- `assembleDebug`
- `lintDebug`
- التوقيع مطابق لشهادة حزمة `2.1.3`.
- الحزمة تحتوي معماريات `arm64-v8a` و`armeabi-v7a` و`x86` و`x86_64`.

ثبت الحزمة فوق التطبيق الحالي دون حذفه. إذا بقي التطبيق لا يفتح، أوصل الجوال بالكمبيوتر مع تصحيح USB وشغّل `adb logcat` للحصول على سبب خاص بالجهاز قبل حذف أي بيانات.
