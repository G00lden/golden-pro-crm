# كتالوج المنتجات المرتبط بسلة

قسم المنتجات داخل CRM يعرض كتالوج سلة ويعامل سلة كمصدر الحقيقة لبيانات البيع:

- الاسم والصورة و`SKU` والأسعار.
- حالة الإتاحة والمخزون.
- التصنيفات والمتغيرات وروابط المنتج في المتجر ولوحة سلة.

تبقى إعدادات الصيانة داخل CRM ولا تستبدلها المزامنة، ومنها مدة الصيانة والقالب والضمان والمهام ووسائط التذكير.

## المزامنة

الزر «مزامنة سلة» يستدعي:

```text
POST /api/integrations/salla/products/sync
```

ويجلب كل صفحات `GET /admin/v2/products` باستخدام صلاحية `products.read`. تعرض الشاشة وقت آخر مزامنة وعدد المنتجات المضافة والمحدثة والمكررة التي تم دمجها.

حدثا `product.created` و`product.updated` يشغلان مزامنة الكتالوج. حدث `product.deleted` لا يحذف التاريخ المرتبط بالمنتج؛ يعلّم السجل بأنه محذوف وغير متاح حتى تبقى التركيبات والحجوزات القديمة سليمة.

## إزالة التكرار

التنظيف يعمل تلقائياً قبل كل مزامنة، ويمكن تشغيله يدوياً عبر:

```text
POST /api/products/deduplicate
```

قواعد الدمج:

1. معرّف منتج سلة `store_product_id` هو المفتاح الأساسي.
2. يستخدم `SKU` لضم سجل يدوي أو قديم عندما يعود إلى منتج واحد فقط في سلة.
3. إذا كان في سلة منتجان مختلفان لهما `SKU` نفسه يبقيهما CRM منفصلين.
4. قبل حذف السجل المكرر يعاد ربط التركيبات والحجوزات والتذكيرات والأجهزة ودورات الصيانة والطلبات والعروض والفواتير بالسجل الأساسي.
5. إعداد الصيانة المفعّل يملك الأولوية على القيم الافتراضية أثناء الدمج.

## التخزين

تدعم بنية SQLite وSupabase وFirestore حقول الصور والتصنيفات والمتغيرات والروابط والحالة والمخزون. أضيفت ترقية Supabase في:

```text
supabase/migrations/20260713130000_salla_product_catalog.sql
```
