## الآثار الجانبية المعروفة

- ربط مفتاح SSH عبر Contabo أعاد تشغيل الخادم مرة واحدة، فحدث انقطاع إدارة وخدمة قصير.
- إعادة إنشاء حاوية CRM أحدثت انقطاعاً قصيراً آخر أثناء انتقال الصورة، ثم عادت الصحة إلى `healthy`.
- الرسائل والحملات الآلية متوقفة عمداً بعد النشر. إعادة تفعيلها تحتاج اعتماد مزود WhatsApp والقوالب ثم تغيير بوابات الأمان صراحة.
- مزامنة سلة المجدولة بقيت فعالة؛ لذلك يمكن أن تتغير أعداد العملاء والطلبات لاحقاً بسبب بيانات حقيقية، لا بسبب الترحيل.
- بناء التطوير أظهر ست ملاحظات متوسطة في تبعيات التطوير، بينما طبقة التشغيل النهائية أبلغت عن صفر ثغرات في فحص `npm` الخاص بها.

## إصلاح مسار النشر

كُشف أثناء التنفيذ أن GNU `tar` على Windows يفسر المسار المطلق الذي يبدأ بحرف القرص كمسار بعيد، وأن PowerShell لا يوقف السكربت تلقائياً عند فشل بعض الأدوات الأصلية. عُدّل `scripts/deploy-vps.ps1` ليقوم بما يلي:

- إنشاء الحزمة عبر مسار نسبي متوافق مع Windows وGNU tar.
- إيقاف النشر فوراً عند فشل `npm` أو `tar` أو `scp` أو `ssh`.
- تطبيع نهايات أسطر سكربتات Linux بعد الرفع لمنع خطأ `pipefail\r`.

هذا الإصلاح يمنع أن يعيد نشر مصدر قديم بصمت عندما تفشل الحزمة المحلية.
