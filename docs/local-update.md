# تحديث crm.breexe-pro.com (سيتب محلي + Cloudflare Tunnel)

الموقع يعمل على **جهاز محلي** ومربوط بالدومين عبر **Cloudflare Tunnel** (`cloudflared`).
مافيش VPS ولا نشر تلقائي عبر GitHub — GitHub لا يقدر يوصل لجهازك المحلي.

## ليه أحيانًا "ما يتغير شي" بعد التعديل؟

لأن السيرفر الشغّال على جهازك **محمّل الكود القديم في الذاكرة**. تعديل الكود على GitHub وحده لا يكفي —
لازم:
1. **تسحب** الكود الجديد على جهازك (`git pull`)، و
2. **تعيد تشغيل** السيرفر عشان يقرأ الكود الجديد.

## الطريقة السهلة — ضغطة واحدة

اعمل **دبل-كليك** على `update-local.cmd` في مجلد المشروع. هو يعمل كل شيء:
`git pull` → `npm install` → `npm run build` → يوقف السيرفر القديم → يشغّل الجديد + يتأكد أن النفق شغّال.

بعدها افتح `crm.breexe-pro.com` واعمل **Ctrl+Shift+R** (لتجاوز كاش المتصفح).

## الطريقة اليدوية

```powershell
cd <مجلد المشروع>
git pull origin main
npm install
npm run build
# أقفل نافذة السيرفر القديم، ثم:
npm run ops:start-local
```

## تحقّق أن التحديث اشتغل

```powershell
curl http://127.0.0.1:3000/api/health
```
لو رجّع `ok` فالسيرفر الجديد شغّال. جرّب تصدير فاتورة جديدة للتأكد من ظهور التعديلات.

## تحديث تلقائي على جهازك (مُفعّل عبر مهمة مجدولة)

اعمل **دبل-كليك مرة واحدة** على `register-auto-update.cmd` (يُفضّل كليك يمين > Run as administrator).
هذا يسجّل مهمة مجدولة في ويندوز اسمها **"Breexe Pro CRM Auto Update"** تشتغل:
- **عند تسجيل الدخول**، و
- **كل 60 دقيقة**.

في كل مرة: تسحب آخر كود من main. **لو فيه جديد** تبني وتعيد التشغيل تلقائيًا؛
**لو مافيش جديد** تتأكد فقط أن السيرفر والنفق شغّالين (بدون إعادة تشغيل / بدون انقطاع).
يعني بعد التفعيل، أي تعديل يندمج في main ينزل على الموقع لوحده خلال ساعة كحد أقصى — بدون أي تدخّل.

### تخصيص أو إلغاء
- فترة مختلفة (مثلاً كل 15 دقيقة):
  ```powershell
  powershell -ExecutionPolicy Bypass -File scripts\register-update-task.ps1 -IntervalMinutes 15 -RunNow
  ```
- إلغاء التحديث التلقائي:
  ```powershell
  Unregister-ScheduledTask -TaskName "Breexe Pro CRM Auto Update" -Confirm:$false
  ```

> ملاحظة: التحديث التلقائي يعيد تشغيل السيرفر لثوانٍ عند وجود كود جديد فقط، فالانقطاع لحظي ونادر.
> لو تفضّل التحكم اليدوي الكامل، تجاهل التفعيل التلقائي واستخدم `update-local.cmd` وقت ما تحب.
