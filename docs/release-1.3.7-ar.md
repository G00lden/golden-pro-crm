# الإصدار 1.3.7 — تكامل سلة وسلامة الفواتير

تاريخ الإصدار: 2026-07-13

## ما تغيّر

- صُححت معادلات عروض الأسعار والفواتير عند الجمع بين السعر الشامل للضريبة والخصم والرسوم الإضافية، وأصبحت المعاينة والطباعة ورمز QR والتحويل من عرض إلى فاتورة تستخدم المجموع نفسه.
- أضيف حقل مستقل للرسوم الإضافية بعد الضريبة، مع ترقية آمنة لقاعدتي SQLite وSupabase.
- أصبحت تواريخ طلبات سلة محفوظة كما يرسلها المتجر، مع المنطقة الزمنية، والفرز حسب تاريخ ووقت الطلب.
- أضيفت فلاتر موسعة للعملاء والطلبات تشمل المصدر والتاريخ والحالة والدفع والشحن والمدينة والقناة والموظف والفرع والوسوم والمنتج والقيمة والقراءة.
- حُسنت مزامنة كتالوج سلة وإزالة التكرار وربط المنتج بالمعرف البعيد، مع منع حذف مرجع ما زال مستخدمًا.
- أضيف تحديث جماعي لحالات طلبات سلة مع تأكيد، ومنع الضغط المتكرر، والإبقاء على الطلبات الفاشلة محددة لإعادة المحاولة.
- وُحدت سياسة إرسال واتساب: وضع المحاكاة لا يطلب كودًا ولا يرفع عداد التذكير ولا يغيّر حالة المستند أو يدّعي الإرسال.
- أصبحت حقول قوالب واتساب المطلوبة ظاهرة ومتحققًا منها قبل الإرسال.
- فُرضت صلاحيات الأدوار في الواجهة والخادم؛ إدارة المستخدمين وواتساب للمسؤول، والحملات والمكالمات للمسؤول والمدير.
- مُنعت بيانات التجربة في الإنتاج، وأزيلت مربعات صلاحيات مخصصة كانت مخزنة بلا تطبيق فعلي.
- أضيف استقبال آمن للعملاء المحتملين من صفحات الموقع، ومسار قياس محدود الحقول لا يخزن بيانات شخصية.
- أضيفت بيئة اختبار معزولة وقابلة للتكرار لا تقرأ أسرار الإنتاج ولا تتصل بسلة أو واتساب.
- رُبط رقم التواصل العام بعقد بناء Docker، وأصبح النشر يرفض رقمًا مفقودًا أو غير صالح بدل إنتاج أزرار اتصال وواتساب معطلة.
- أصبح Caddy وحده يمرر عنوان العميل الموثوق إلى الخادم، مع تجاهل ترويسات العميل المزورة.
- أصبح نشر VPS يحفظ الصورة والبيئة وملفي Compose وCaddy الفعليين قبل الاستبدال، ويفحص الإصدار داخليًا وعبر HTTPS، ثم يعيد الحالة السابقة كاملة عند الفشل.
- أُوقف مسار Cloud Run القديم لأنه لا يوفّر التخزين الدائم الذي يحتاجه SQLite وجلسة واتساب؛ مسار التشغيل المدعوم هو VPS.

## الآثار الجانبية المقصودة

- كل إرسال يدوي ينفذ فحصًا قصيرًا لسياسة الإرسال قبل الطلب، لذلك قد تزيد الاستجابة بزمن طفيف.
- المحاكاة تُسجل كسجل تدقيق `dry_run`، لكنها لا تُحسب رسالة فعلية ولا تغيّر مرحلة التذكير.
- المستخدم ذو دور محدود لن يرى الصفحات المحمية، كما سيرفض الخادم روابطها المباشرة بالحالة `403`.
- تشغيل الترقية على قاعدة إنتاج قديمة ينشئ نسخة احتياطية تلقائية قبل تعديل المخطط.
- حذف المنتج اليدوي أصبح تعطيلًا منطقيًا؛ يبقى السجل مخفيًا في التخزين لحماية مراجع الطلبات، ولذلك قد ينمو حجم جدول المنتجات تدريجيًا.
- فشل فحص Caddy أو HTTPS أثناء النشر يطيل زمن العملية بسبب إعادة إنشاء الإصدار السابق وفحصه، لكنه يمنع بقاء النظام في حالة نصف محدثة.

## قيد تشغيلي معروف

مزامنة سلة الكاملة تحتاج خروجًا شبكيًا تقبله واجهة Salla API. عنوان خادم VPS الحالي محجوب إقليميًا من سلة؛ لذلك يلزم اعتماد عنوان الخادم لدى سلة أو تمرير الاتصال عبر مخرج سعودي/خليجي خاص وآمن. لا يجوز استخدام وكيل عام لأن الاتصال يحمل رموز OAuth للمتجر.
